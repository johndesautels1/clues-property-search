  Current Code Reality (search.ts ~line 4894)

  const llmCalls = [
    { id: 'perplexity-a', fn: callPerplexityPromptA, ... },
    { id: 'perplexity-b', fn: callPerplexityPromptB, ... },
    { id: 'perplexity-c', fn: callPerplexityPromptC, ... },
    { id: 'perplexity-d', fn: callPerplexityPromptD, ... },
    { id: 'perplexity-e', fn: callPerplexityPromptE, ... },
    { id: 'gemini', fn: callGemini, ... },
    { id: 'gpt', fn: callGPT5, ... },
    { id: 'grok', fn: callGrok, ... },
    { id: 'claude-sonnet', fn: callClaudeSonnet, ... },
    { id: 'claude-opus', fn: callClaudeOpus, ... },
  ];

  // PROBLEM: Currently fires ALL in parallel
  const results = await Promise.all(llmCalls.map(call => call.fn(address)));

  The Problem

  If Grok returns in 2 seconds with 12_market_value_estimate = $450,000
  And Perplexity returns in 5 seconds with 12_market_value_estimate = $475,000

  Current behavior: Whichever result gets merged LAST wins. This is random based on network timing.

  The Fix (Simple)

  Option A: Sequential Firing (Recommended)
  // Fire in ORDER, stop searching field once found
  let fields = {};

  // Tier 4 - Primary
  const ppxResults = await callPerplexityAll(address);
  fields = mergeFields(fields, ppxResults); // First write wins

  const gemResults = await callGemini(address);
  fields = mergeFieldsIfMissing(fields, gemResults); // Only fills gaps

  // Tier 5 - Secondary (only if fields still missing)
  if (countMissingFields(fields) > 0) {
    const gptResults = await callGPT5(address);
    fields = mergeFieldsIfMissing(fields, gptResults);
  }
  // ... continue

  Option B: Parallel with Priority Merge
  // Fire all in parallel
  const [ppx, gem, gpt, grok, sonnet, opus] = await Promise.all([...]);

  // Merge in PRIORITY ORDER (higher tier always wins)
  let fields = {};
  fields = mergeFields(fields, opus);    // Lowest priority - written first
  fields = mergeFields(fields, sonnet);  // Overwrites opus
  fields = mergeFields(fields, grok);    // Overwrites sonnet
  fields = mergeFields(fields, gpt);     // Overwrites grok
  fields = mergeFields(fields, gem);     // Overwrites gpt
  fields = mergeFields(fields, ppx);     // HIGHEST - overwrites all

  My Recommendation: Option A (Sequential)

  - Simpler to understand
  - Lower cost (stop early if fields found)
  - No timing race conditions
  - Clear audit trail of which LLM provided which field
